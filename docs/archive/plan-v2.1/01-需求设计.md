## 你的角色与目标
你是本迭代的**需求/设计负责人**。目标是把 v2.1 新增的三项功能（需求 10/11/12）从产品语言拆解为**开发可直接落地的设计规格**，输出：
1. 各需求的**交互流程图/状态机**（文字或 mermaid）；
2. 新增/修改的**数据模型字段清单**与**默认值**；
3. 与既有 `PlanItem` / `TrainingPlan` / `PlanTemplate` 模型的**兼容性说明**；
4. 对下游（数据层、前端、测试）的**接口契约建议**（方法签名草案、字段命名）。

## 三项功能的设计要点（必须遵守）
### 需求 10 — 自定义训练计划
- 两种组建方式共用同一编辑器：**选模板再改**（用 `generatePlanFromTemplate` 产物作初始草稿）、**空白自建**（无预填，先选目标/难度再选方法+排期）。
- 自定义阶段**仅允许调整**：训练方法（增删/替换）、每周训练天数、具体训练日期。
- **不得**在自定义阶段暴露「单次时长 / 强度 / 周期长度」入口（留待需求 11）。
- 「我的模板」需设计为可持久化的用户数据：`UserPlanTemplate` 以 `days: [UserPlanTemplateDay]`（每日 `dayOffset` + `methodIds`）承载**每日可多方法（Q3）**；为兼容需求 10 字段命名，仍保留 `methodIds` / `trainingDayOffsets` 作为计算属性。字段至少含 `name / difficulty / frequency / goal / icon / description? / days`。

### 需求 11 — 当前计划逐条编辑
- 编辑态为**草稿**（内存可变副本），取消则不落库。
- 可操作：替换方法、改单次时长、改所在日期、增删某天项目。
- 保存校验：时长 > 0；日期 ∈ `[plan.startDate, plan.endDate]`；保存后调用 `updateProgress()`。

### 需求 12 — 今日训练动作直达陪练
- 点击动作行（非完成按钮）→ 动作详情页（复用 `TrainingDetailView` 方法说明以满足 AC-C.2/AC-C.5）。
- 详情页「开始陪练」→ 进入既有 `CoachView`（携带 `planItemId` 与方法参数）。
- 陪练**正常完成**（生成训练记录）后回调 `markPlanItemCompleted(planItemId)`；中途自行退出（无记录）**不**勾选；已 `isCompleted` 项点击仅查看。

## 约束（来自 requirements.md）
- 纯离线、iOS 16+；目标用户 35–55 岁：操作 ≤ 2 次点击、可点击区 ≥ 44pt、Dynamic Type、VoiceOver（AC-NF.4/5/6）。
- 合规：方法说明须含来源与禁忌（AC-C.2/AC-C.5）；「我的模板」排除 iCloud 备份（AC-7.5/AC-NF.7）。
- 不引入新三方库、新架构模式；沿用既有 MVVM + Repository + Core Data。
