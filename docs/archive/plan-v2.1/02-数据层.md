## 你的角色与目标
你是本迭代的**数据层负责人**。负责为需求 10/11/12 落地本地持久化与规则扩展：

2. 为逐条编辑新增单条计划项增删改能力；

## 具体任务
### 1. 「我的模板」模型与持久化（需求 10 / AC-10.5）
- 在 `PlanModels.swift` 新增领域模型：
  ```swift
  struct UserPlanTemplateDay: Codable, Identifiable, Hashable {
      let id: UUID
      var dayOffset: Int       // 0...6
      var methodIds: [UUID]     // 该日方法（≥1，Q3 支持一日多方法）
  }

  struct UserPlanTemplate: Identifiable, Codable {
      let id: UUID
      var name: String
      let difficulty: DifficultyLevel
      let frequency: Int
      let goal: TrainingGoal
      let icon: String
      let days: [UserPlanTemplateDay]   // 每日方法（Q3 支持一日多方法）
      var description: String?          // Q5 确认：可选
      let createdAt: Date
      var updatedAt: Date
      var methodIds: [UUID] { /* 全部方法去重，兼容字段命名 */ }
      var trainingDayOffsets: [Int] { days.map { $0.dayOffset }.sorted() }
  }
  ```
- 在 `*.xcdatamodeld` 新增 `CDUserPlanTemplate` 实体：属性 `id(UUID/String)`、`name(String)`、`difficulty(Int16)`、`frequency(Int16)`、`goal(Int16)`、`icon(String)`、`daysData(Binary，JSON 编码 [UserPlanTemplateDay])`、`desc(String?)`、`createdAt/updatedAt(Date)`。排除 iCloud 备份（呼应 AC-7.5）。
- 在 `PlanRepository`（或新建 `UserTemplateRepository`）提供 CRUD：`saveUserTemplate` / `fetchUserTemplates` / `deleteUserTemplate`。

### 2. 逐条编辑（需求 11 / AC-11.4/11.5/11.6）
- **Q6 已确认：仅全量保存**。不新增单条粒度 Repository 方法（`addPlanItem`/`updatePlanItem`/`removePlanItem` 不实现）；编辑态的增/删/改均在内存 `PlanEditDraft.items` 上完成，保存时调用既有 `updatePlanItems(planId:items:)` 一次性整体落库。
- `updatePlanItems` 实现须走**后台上下文 + 主上下文合并**（既有 `performBackgroundTask` 模式），保存后调用 `TrainingPlan.updateProgress()` 并重算（呼应 AC-NF.8）。
- 不改 `markPlanItemCompleted` / `updatePlanItems` / `adjustPlanIfNeeded` 既有行为。

### 3. PlanService 扩展（需求 10 / AC-10.2/10.3/10.6）
- 新增 `buildCustomPlan(dayDrafts: [DayDraft], baseTemplate: PlanTemplate? = nil, goal: TrainingGoal? = nil) -> TrainingPlan`（Q3 每日可多方法）：对每个 `day ∈ dayDrafts` 的每个 `methodId` 生成一条 `PlanItem(date: methodId: methodName: duration: method.defaultDuration)`；固定 7 天周期（呼应 AC-10.4 不暴露周期）。
- 新增 `allTemplatesForSelection() -> [PlanTemplate]`：合并预设（`planTemplates()`）与「我的模板」映射出的 `PlanTemplate`（供选择器展示）。
- 新增 `saveUserTemplate(_:)` / `loadUserTemplates() -> [UserPlanTemplate]`（封装 Repository）。

## 约束
- 纯离线；不引入新三方库；保持既有 Repository 错误风格（沿用 `print(...)` 错误日志，不打印方法/计划敏感详情）。
- 所有写操作后台上下文执行，主线程只刷新；保证 `updateProgress` 结果与 `items` 一致。
- 「我的模板」属用户数据，Core Data 存储且排除 iCloud 备份。

## 验收门槛
- `UserPlanTemplate` 可保存→读取→删除，且重新启动后仍在（持久化正确）。
- 单条增删改后 `plan.items` 与 `progress` 一致，并触发视图刷新。
- `buildCustomPlan` 对「选模板」与「空白」两种输入均生成合法 `TrainingPlan`（训练日均在计划周期内、方法来自 `allTrainingMethods()`）。
- 不破坏既有 `regeneratePlan` / `adjustPlanIfNeeded` / `createPlanFromTemplate` 行为（回归通过）。
