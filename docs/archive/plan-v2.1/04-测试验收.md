# 工作流分工 Prompt — 04 测试与验收

> 本文件为「男性控制训练」iOS App **v2.1 训练计划增强**迭代中「测试与验收」角色的下游 AI 执行 prompt。请基于 requirements.md 的 AC 编号编写可执行的测试，并输出覆盖率追溯。

## 你的角色与目标
你是本迭代的**测试/验收负责人**。负责为需求 10/11/12 的全部 AC 编写测试并给出验收结论：
1. 单元测试覆盖 `PlanService.buildCustomPlan`、Repository 增删改、我的模板 CRUD、进度重算；
2. UI/集成测试覆盖自定义入口、编辑草稿取消/保存、今日动作跳转与自动勾选；
3. 输出 **AC 覆盖矩阵**（每条 AC → 测试/手动验证结论）。

## 测试任务清单（按 AC 映射）
### 需求 10
- AC-10.2/10.3：`buildCustomPlan` 对「选模板」与「空白」输入均生成合法 `TrainingPlan`；训练日落在周期内、方法来自 `allTrainingMethods()`。
- AC-10.4：非法维度（如尝试设置时长/强度）在视图层不应暴露——以代码审查 + UI 测试断言无对应控件。
- AC-10.5：`UserPlanTemplate` 保存→读取→删除往返正确（Core Data 持久化，重启后仍在）。
- AC-10.6：覆盖活跃计划前触发二次确认；写入后 `progress` 由 `updateProgress()` 重算。
- AC-10.1/10.7：入口 ≤ 2 次点击、Dynamic Type/VoiceOver——以 UI 测试 + 审查覆盖（无障碍多为主动验证项，标注为手动/UI 验证）。

### 需求 11
- AC-11.2/11.3：`addPlanItem` / `updatePlanItem` / `removePlanItem` 单测：增删改后 `plan.items` 与 `progress` 一致。
- AC-11.4：保存走后台上下文、主上下文合并、视图刷新（单测 + 集成）。
- AC-11.5：时长 ≤ 0、日期越界的保存被拒绝并提示（单测断言校验逻辑）。
- AC-11.6：既有 `regeneratePlan` / `adjustPlanIfNeeded` / `createPlanFromTemplate` 回归不受影响。

### 需求 12
- AC-12.1/12.2：今日动作行点击进入 `PlanItemDetailView`（非完成按钮）；详情页含方法说明与「开始陪练」（UI 测试）。
- AC-12.3/12.4：开始陪练携带 `planItemId`；模拟陪练完成后 `markItemCompleted(planItemId)` 被调用且视图刷新；中途退出不勾选（集成测试/单元测试 mock Coach 完成回调）。
- AC-12.5：已 `isCompleted` 项点击仅查看，不重复触发/勾选。
- AC-12.6：入口无障碍（点击区 ≥ 44pt、VoiceOver 标签）——审查 + 手动验证标注。

## 约束
- 纯离线；不引入新三方测试库（沿用项目既有测试框架，如 XCTest）；不打印敏感数据。
- 量化验收：沿用 requirements.md §2.2 量化约定（如可点击区、点击次数）。

## 交付物
- `tests-需求10-自定义计划.swift` / `tests-需求11-逐条编辑.swift` / `tests-需求12-直达陪练.swift`（或并入既有测试 target）。

## 验收门槛
- 需求 10/11/12 的每条 AC 在 `AC覆盖矩阵.md` 中至少有一条对应验证，不得遗漏。
- 单测在 CI/本地可运行通过；无法自动化的（无障碍、真机）明确标注为手动验证且给出操作步骤。
- 回归：既有需求 3 的计划生成/智能调整未被破坏（相关既有测试仍通过）。
