## 1. 问题 A：GitHub 测试失败（全部源于 Keychain）

### 1.1 失败清单（日志实测）

| 套件 | 失败样例 |
|------|----------|
| `ModelTests` | `testKeychainSaveAndLoad`、`testKeychainCodableObject`、`testKeychainDelete`、`testEncryptDecryptString` 等（21 例中约 9 失败） |
| `PerformanceTests` | `testDecryptDataPerformance`（"加密失败"） |
| `SecurityServiceTests` | `testSetPassword`、`testAuthModeAfterSettingPassword`、`testEncryptDecryptRoundTrip`、`testEncryptEmptyData`、`testEncryptLargeData` 等 |

### 1.2 根因（已读源码坐实）

1. `KeychainService.save` 使用 `kSecAttrAccessibleWhenUnlockedThisDeviceOnly`。该 `ThisDeviceOnly` 属性依赖**设备级密钥库**，在 **iOS 模拟器 / CI** 环境下 `SecItemAdd` 常保存失败，返回 `false`。
2. 测试直接调用 `KeychainService.shared`（真实 Keychain，无 mock）。链式级联：
   - `KeychainService.save` 失败 → `CryptoService.getOrCreateKey()` 存不住密钥 → `encrypt` 返回 `nil`（"加密失败"）；
   - `setPassword` 的 `saltSaved && passwordSaved` 为 `false` → `isPasswordSet` 不置位 → 认证模式落为 `"none"`（与期望 `"password"` / `"biometricAndPassword"` 不符）。
3. 属**环境性失败**，非本轮引入的回归——此前构建一直失败、测试从未真正跑通，故本次构建成功后才首次暴露。

## 2. 问题 B：LiveContainer 安装签名失败

报错：`Failed to Sign .../ControlTraining.app/PlugIns/ControlTrainingTests.xctest.dSYM/Contents/Resources/DWARF/ControlTrainingTests`

### 2.1 根因（已读 `build-ipa.yml` + `project.yml` 坐实）

- `build-ipa.yml` 用 `xcodebuild build -scheme ControlTraining` 构建后，把 `.app` 直接 `cp` + `zip` 成 IPA。
- `project.yml` 的 scheme `build.targets` 把 `ControlTrainingTests` / `ControlTrainingUITests` 一并列入 → 测试包被编入 App 的 `PlugIns/`。
- LiveContainer 安装时需对包内所有内容重签名，但 `.xctest.dSYM`（调试符号）无法签名 → 安装失败。
- 系统框架嵌入问题已在上一轮（v5.7 第四次）修复，与本问题无关。

## 3. 拟执行的修复方案（待确认）

### 修复 A — Keychain 测试环境降级（改 `KeychainService.swift`）

在 `KeychainService` 中检测 XCTest 环境，命中时改用**内存字典**存储，绕过模拟器 Keychain：

```swift
private var isTestEnvironment: Bool {
    ProcessInfo.processInfo.environment["XCTestConfigurationFilePath"] != nil
        || NSClassFromString("XCTestCase") != nil
}
private var memoryStore: [String: Data] = [:]
// save / load / delete 在 isTestEnvironment 命中时读写 memoryStore
```

- 生产环境（无 XCTest）仍走真实 Keychain（`ThisDeviceOnly` 安全等级不变），**不影响线上行为**。
- 备选：将 `KeychainService` 抽为 protocol + 测试注入 mock（改动大、回归风险高），本次不推荐。

### 修复 B — 发布包不再编入测试包（改 `project.yml` + `build-ipa.yml`）

1. `project.yml` scheme 调整：`build.targets` 仅保留 App，测试目标移入 `test.targets`：
   ```yaml
   schemes:
     ControlTraining:
       build:
         targets:
           ControlTraining: all
       test:
         targets:
           - ControlTrainingTests
           - ControlTrainingUITests
   ```
   → `xcodebuild build`（Build IPA）不再编译/嵌入测试包；`xcodebuild test` 仍经 `test.targets` 正常构建运行。
2. `build-ipa.yml` 的「Create IPA」步骤加保险：打包前 `rm -rf` 掉 `.app/PlugIns`，确保 LiveContainer 可签名。
