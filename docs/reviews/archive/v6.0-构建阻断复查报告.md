## 二、逐项核对

### #1 PlanEditView `#Preview` 编译阻断 — ❌ FAIL（待修复）
- **现象**：`PlanEditView.swift:262:5: error: 'buildExpression' is unavailable: this expression does not conform to 'View'`
- **根因**：`#Preview { ... }` 尾随闭包为 `@ViewBuilder`，要求表达式为 `View`；原代码在闭包内 `let vm = PlanViewModel()` 后直接 `vm.editingDraft = PlanEditDraft(...)`——赋值返回 `Void`，不遵守 `View` 协议，触发 `buildExpression` 不可用。
- **影响**：整个 `ControlTraining` app target 编译失败 → 依赖它的 `ControlTrainingTests`、`ControlTrainingUITests` 均无法构建，测试被整体取消，`xcodebuild test` 报告 **0 用例执行、4 failures（均为 build 失败）**。
- **修复建议**（最小、仅改 preview、无逻辑改动）：将带副作用的赋值收进返回 `PlanViewModel` 的闭包：
  ```swift
  #Preview {
      let vm: PlanViewModel = {
          let vm = PlanViewModel()
          let items = [
              PlanItem(date: Date(),
                       methodId: TrainingContentData.allTrainingMethods()[0].id,
                       methodName: TrainingContentData.allTrainingMethods()[0].name,
                       duration: 300)
          ]
          vm.editingDraft = PlanEditDraft(
              planId: UUID(),
              startDate: Date(),
              endDate: Calendar.current.date(byAdding: .day, value: 7, to: Date())!,
              items: items)
          return vm
      }()
      PlanEditView(viewModel: vm)
  }
  ```
- 本缺陷属实现侧 `#Preview` 误用，与本迭代测试文件无关。
