## 2. 残留编译错误（构建仍将失败）

即便 RC-1/2/4/5/6/7a/7c/8 全部生效，以下 **2 个错误**仍导致 `xcodebuild build` 中止：

### 残留 1 — RC-7b 未修复（原错误）
- `OnboardingView.swift:382` — `Image(systemName: goal.icon)`：`value of type 'TrainingGoal' has no member 'icon'`
- 实证：`OnboardingView.swift:375` 声明 `let goal: TrainingGoal`；`TrainingGoal` 定义于 `PlanModels.swift:123`（仅 4 个 `case`，无 `icon`）。编码 AI 把 `icon` 加到了 `TrainingCategory`（`TrainingModels.swift:14`），与目标类型无关。
- 修复：在 `TrainingGoal` 枚举内增加 `var icon: String { switch ... }`（如 `endurance → "figure.run"` 等），或把 `goal.icon` 改为基于现有字段的映射。

### 残留 2 — RC-3 新回归（修复引入）
- `SecurityService.swift:307` — `PBKDF2<SHA256>.deriveKey(password:password, salt:salt, rounds: 100_000)`
- 实证：CryptoKit 的 `PBKDF2<Hash>.deriveKey` 参数签名为 `password:salt:iterations:keyLength:`，**参数名为 `iterations:` 而非 `rounds:`**。因此 `rounds: 100_000` 会触发 `incorrect argument label in call (have 'password:salt:rounds:', expected 'password:salt:iterations:')`。
- 修复：将 `rounds: 100_000` 改为 `iterations: 100_000`。
- 附带：`SecurityService.swift:4-5` 出现重复 `import CryptoKit`，非阻断，建议删除一行。

## 4. 方法论提示

- 本轮复审再次证明：**静态逐文件阅读不能替代实编译**。"改一处可能引入另一处"（RC-3 的 `rounds:`），且"修复可能改错对象"（RC-7b 把 `icon` 加到了错误类型）。
- 编码 AI 完成修复后**必须**自行跑一次 `xcodebuild build` 实证，再声明"已修复"，而非仅按清单逐项声明。
- 建议把"修复后实编译"纳入 CI 门禁前的本地自检步骤。
