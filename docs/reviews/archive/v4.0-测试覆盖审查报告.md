## 一、关键发现（变更前根因）

### 发现 1（阻塞性）— 测试从未被构建/运行
`project.yml` 原本**只定义了一个 `ControlTraining` application target，没有测试 target**。
后果：
- `xcodegen generate` 不会生成任何测试 target，159 个用例在工程中根本不被编译；
- `.github/workflows/build-ipa.yml` 仅执行 `xcodebuild build`（且 `-destination generic/platform=iOS` 设备目标），**从不运行 `xcodebuild test`**。

> 结论：此前"136+ 用例"只是源文件计数，**CI 从未实际执行过任何测试**。这是"无集成测试"表象下的真正根因。

### 发现 2（结构性）— 仅有单层单测，无跨层闭环
既有用例均为"注入内存库 → 单类调用 → 断言"的孤立单测，缺少
`Repository（持久化）→ Service（计算）→ ViewModel（反映）` 的端到端数据闭环验证。
例如 `AnalysisServiceTests` 写入 `trainingRepo` 后直接调 `calculateOverallScore()`，
但未验证写入是否真正落盘、是否能被 `HomeViewModel.loadData()` 反映。

### 发现 3 — 无任何 UI 测试
无 `XCUI` 测试，首启引导、Tab 导航、关键 ≤2 次点击路径（AC-NF.5 / AC-1.4）的可达性无自动化保障。
