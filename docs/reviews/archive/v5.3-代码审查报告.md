## 0. 方法论更正（重要）

v5.2（2026-07-11）给出的「✅ 通过 / 主 Target 编译阻断已消除」是**错误结论**。该结论来自**纯静态源码阅读**，从未真正执行 `xcodebuild build`。本次 CI 实编译暴露出 v5.0~v5.2 三轮审查**从未捕获**的 34 个编译错误。

- ✅ 正面事实：日志中**未出现** BUG-CT-08~12 任何一项（5 项 P0），说明 v5.2 对 P0 的修复确实生效。
- ❌ 负面事实：此前审查**只盯 P0、不编译**，导致贯穿 11 个文件的编译错误全部漏检。静态「能编译」≠ 真能编译。

**本报告以 `docs/build.log` 的真实编译器输出为唯一权威证据，并对每条错误在「当前磁盘代码」上的存在性做了独立核实。**

### RC-2 中文直引号截断字符串字面量（5 错误 / 1 文件）✅已存
| 文件:行 | 错误 |
|---------|------|
| `PrivacyPolicyView.swift:17` | expected ',' separator |
| `PrivacyPolicyView.swift:17` | cannot find '本应用' in scope |
| `PrivacyPolicyView.swift:17` | missing argument label 'format:' in call |
| `PrivacyPolicyView.swift:85` | expected ',' separator |
| `PrivacyPolicyView.swift:85` | cannot find '删除所有数据' in scope |

根因：`Text("…以下简称"本应用"）…")` 与 `detail: "…"删除所有数据"…"` 内部使用了**中文直引号 `"`**。Swift 字符串以 `"` 结束，内部直引号提前截断字符串，导致后续中文被当作标识符/参数错误。应改用全角引号「」或转义，或将内部引号改为 `'`。

### RC-3 CommonCrypto 符号不可用（6 错误 / 1 文件）✅已存 —— SEC-01 修复的回归
| 文件:行 | 错误 |
|---------|------|
| `SecurityService.swift:310` | cannot find 'CCKeyDerivationPBKDF' in scope |
| `SecurityService.swift:311` | cannot find 'CCPBKDFAlgorithm' in scope |
| `SecurityService.swift:311` | cannot find 'kCCPBKDF2' in scope |
| `SecurityService.swift:314` | cannot find 'CCPseudoRandomAlgorithm' in scope |
| `SecurityService.swift:314` | cannot find 'kCCPRFHmacAlgSHA256' in scope |
| `SecurityService.swift:319` | cannot find 'kCCSuccess' in scope |

根因：SEC-01（v5.1 修复）将 `hashPassword` 改为 PBKDF2，**但使用了 CommonCrypto 的 C API 却未 `import CommonCrypto`**（文件头仅 `import CryptoKit`，见 `CryptoService.swift:2` 同类导入风格）。C 符号不可见 → 全部 "cannot find in scope"。修复：① 加 `import CommonCrypto`；或 ②（推荐）改用已导入的 `CryptoKit` 的 `PBKDF2<SHA256>` 重实现，避免依赖 C 模块。

### RC-4 SwiftUI `Section` API 误用（2 错误 / 1 文件）✅已存
| 文件:行 | 错误 |
|---------|------|
| `PrivacySettingsView.swift:19` | type '() -> Text' cannot conform to 'View' |
| `PrivacySettingsView.swift:57` | argument 'footer' must precede argument 'content' |

根因：`Section(header: Text("认证方式"), content: { … }) { } footer: { … }` 混用了两种初始化器签名。带 `header:`/`content:` 标签的重载**没有 `footer:` 参数**；正确写法为 `Section { … } header: { … } footer: { … }` 或 `Section(content:header:footer:)`。当前 `content` 闭包后追加 `footer:` 使编译器误判 `content` 返回 `() -> Text`。

### RC-5 重复声明 `SettingsView`（2 错误 / 2 文件）✅已存
| 文件:行 | 错误 |
|---------|------|
| `ProfileView.swift:162` | invalid redeclaration of 'SettingsView' |
| `SettingsView.swift:47` | ambiguous use of 'init()'（`NavigationStack { SettingsView() }`） |

根因：`SettingsView.swift:4` 与 `ProfileView.swift:162` **各声明了一个 `struct SettingsView: View`**，导致重复声明；`SettingsView.swift:47` 的 `SettingsView()` 调用因此产生 `init()` 歧义。修复：将 `ProfileView.swift:162` 的结构体重命名为 `ProfileSettingsView`（或删除重复定义）。

### RC-6 `Data` ↔ `String` 类型不匹配（2 错误 / 1 文件）✅已存
| 文件:行 | 错误 |
|---------|------|
| `DataViewModel.swift:61` | cannot convert value of type 'Data' to expected argument type 'String' |
| `DataViewModel.swift:110` | cannot convert value of type 'Data' to expected argument type 'String' |

根因：`CryptoService` 的 `encrypt(_: String) -> String?`（:41）与 `decrypt(_: String) -> String?`（:58）收 `String`，但 `DataViewModel` 传入 `Data`（`jsonData` / `sealedBox`）。`CryptoService` 已提供 `encryptData(_: Data) -> Data?`（:76）与 `decryptData(_: Data) -> Data?`（:91）。修复：改用 `encryptData` / `decryptData`。

### RC-7 调用不存在的成员 / 方法（5 错误 / 4 文件）✅已存
| 文件:行 | 错误 |
|---------|------|
| `DataManagementView.swift:152` | cannot call value of non-function type 'Bool'（`viewModel.showError("…")` 解析为 Bool 属性） |
| `OnboardingView.swift:382` | value of type 'TrainingGoal' has no member 'icon' |
| `SettingsViewModel.swift:63` | NotificationService has no member 'requestAuthAndScheduleDefaults' |
| `SettingsViewModel.swift:65` | NotificationService has no member 'cancelAll' |
| `SettingsViewModel.swift:72` | SecurityService has no member 'setBiometricEnabled' |

根因：
- `DataViewModel.swift:17` 有 `@Published var showError = false`（属性），`:217` 又有 `private func showError(_:)`（方法），调用 `viewModel.showError("…")` 解析为 Bool 值 → 无法调用。修复：属性改名（如 `showingError`）或方法改名。
- `TrainingGoal` 类型无 `icon` 成员（`OnboardingView.swift:355` 的 `var icon` 属于 View 而非 `TrainingGoal`）。修复：给 `TrainingGoal` 增加 `icon` 字段或改用现有映射。
- `NotificationService` 实际方法为 `cancelAllNotifications()`（:277），无 `cancelAll` / `requestAuthAndScheduleDefaults`；`SecurityService` 无 `setBiometricEnabled`。修复：实现缺失方法或改调现有 API。

### RC-8 `CoreDataStack` 编译错误（log 4 错误 / 磁盘仅剩 1）⚠️部分已修
| 文件:行 | 错误 | 磁盘状态 |
|---------|------|----------|
| `CoreDataStack.swift:100` | attribute 'private' can only be used in a non-local scope | ⚠️已修（ARC-03 已把方法移出闭包至类级） |
| `CoreDataStack.swift:111` | expected '}' in class | ⚠️已修（括号已闭合） |
| `CoreDataStack.swift:31` | cannot find 'excludeFromiCloudBackup' in scope | ⚠️已修（方法已归位） |
| `CoreDataStack.swift:19` | cannot convert value of type 'String' to type 'NSNumber' in coercion | ✅**仍存** |

根因（残留项:19）：`description.setOption(FileProtectionType.complete.rawValue as NSNumber, forKey:)` 中 `FileProtectionType.complete.rawValue` 为 `String`，`as NSNumber` 强转失败（字符串不能转 `NSNumber`）。`NSPersistentStoreFileProtectionKey` 的 option 值本就是 `String`，应直接传 `FileProtectionType.complete.rawValue`（去掉 `as NSNumber`）。

> 注：RC-8 的 100/111/31 三项在 CI 构建版本中存在（括号错位），**当前磁盘已随 ARC-03 修复**，但 :19 的 NSNumber 错误仍在，故该文件当前仍无法编译。
