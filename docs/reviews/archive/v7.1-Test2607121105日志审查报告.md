## 三、根因分析(合并两日志)

### A. 同一根因:`DayDraft.id` / `UserPlanTemplateDay.id` 无默认值(贯穿 app + tests)

`PlanModels.swift`:
```swift
struct DayDraft: Identifiable, Hashable {           // :86
    let id: UUID                                     // :87  ← 无默认
    var dayOffset: Int
    var methodIds: [UUID]
}
struct UserPlanTemplateDay: Codable, Identifiable, Hashable { // :130
    let id: UUID                                     // :131  ← 无默认
    var dayOffset: Int
    var methodIds: [UUID]
}
```
所有省略 `id:` 的调用都失败。**这同时解释了两日志的全部 `missing argument for parameter 'id'` 错误**——app 侧 5 处、test 侧 6 处(共 11 处)。

- app 侧:`PlanBuilderView:469/499`、`PlanService:438/450`(`DayDraft`)、`PlanViewModel:200`(`UserPlanTemplateDay`)
- test 侧:`tests-需求10` 行 71/72/168/195/196/227

### B. 独立 app 错误:`saveCurrentDraftAsTemplate` 漏 `name:` 标签

`PlanViewModel.swift:197` 定义为 `func saveCurrentDraftAsTemplate(name: String)`,形参带 `name:` 标签;但 `PlanBuilderView.swift:68` 写为 `viewModel.saveCurrentDraftAsTemplate(name)`(漏标签),报 `missing argument label 'name:'`。这是 app 专属 bug,测试日志未涉及。

### C. 测试侧独立错误:`PlanDraft` 成员初始化器参数顺序

`tests-需求10` 行 74/169/228 写 `PlanDraft(dayDrafts: [...], goal: .control, difficulty: .beginner)`,把 `dayDrafts` 放在 `goal` 前。`PlanDraft`(PlanModels.swift:95-100)合成成员初始化器要求按声明顺序 `sourceTemplateId→name→goal→difficulty→dayDrafts`,故报 `argument 'goal' must precede argument 'dayDrafts'`。**仅测试文件涉及**(app 不以此形式调用 `PlanDraft`)。

### D. `tests-需求12` 的 6 条报错 = 级联假错

经核对 `tests-需求12` 所有 API 用法(`TrainingMethod`/`CoachViewModel`/`PlanItemDetailView`/`PlanViewModel`)**均匹配当前签名**,无独立错误。其 `no member 'repo'`/`Double?→Double` 是三个测试文件同批编译、因 `tests-需求10` 硬错误污染共享类型推断的次级假错,修复 A/C 后应自动消失。

### E. Debug/Release 不一致说明(供留意)

两日志同时间戳 `2607121105`,但 Debug 测试运行到达了 test target 阶段(说明其 app 产物在该次增量/缓存环境下"通过"),而 Release 全量编译如实失败。直接读源码已证实上述 6 个 app 错误在当前 `97ca7c2` 中真实存在,**不影响结论:app 源码是坏的,且是 IPA 构建的真正阻断点**。

## 四、修复建议(给编码 AI)—— 改为「先改模型/API,再补测试」

### 主修复(改模型,一处治本,同时修 app + tests 的 11 处 `id` 错误)

在 `PlanModels.swift`:
```swift
struct DayDraft: Identifiable, Hashable {
    let id: UUID = UUID()          // ← 加默认,修复 PlanBuilderView:469/499 + PlanService:438/450 + tests-需求10(6 处)
    ...
}
struct UserPlanTemplateDay: Codable, Identifiable, Hashable {
    let id: UUID = UUID()          // ← 加默认,修复 PlanViewModel:200 + tests-需求10(2 处)
    ...
}
```
> 加默认值不影响 `Identifiable`/`Hashable`/`Codable`:解码时已有 `id` 的模板照常读取;缺省时自动生成。这是最稳、最小侵入的修法。

### 修复 B:`PlanBuilderView.swift:68`
```swift
viewModel.saveCurrentDraftAsTemplate(name: name)   // ← 补 name: 标签
```

### 修复 C:`tests-需求10` 的 3 处 `PlanDraft` 调用改声明顺序
```swift
PlanDraft(goal: .control, difficulty: .beginner,
          dayDrafts: [ DayDraft(id: UUID(), dayOffset: 0, methodIds: [method.id]),
                       DayDraft(id: UUID(), dayOffset: 3, methodIds: [method.id, firstMethod2().id]) ])
```
(将 `goal`/`difficulty` 提到 `dayDrafts` 前;行 69-76、167-171、226-229)

> 注: 主修复已让 `DayDraft(id:...)` 可选,故 C 节里 `DayDraft(id: UUID(), ...)` 也可简写为 `DayDraft(dayOffset: ..., methodIds: ...)`。保留显式 `id` 亦可。
