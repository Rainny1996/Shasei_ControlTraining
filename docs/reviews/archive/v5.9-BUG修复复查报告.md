## 二、逐项核对

### #1 评估完成不跳转 — ✅ PASS
- `AssessmentViewModel.swift:24` 新增 `@Published var shouldDismiss`，并在 `submitAssessment()` 末尾置 `shouldDismiss = true`。
- `AssessmentView.swift:7` 注入 `@Environment(\.dismiss)`，`L61-65` 经 `.onChange(of: viewModel.shouldDismiss)` 调用 `dismiss()`。
- 逻辑闭环正确，sheet 可正常关闭。

> 轻微提示（非阻断）：`onChange(of:_:)` 单参闭包在 iOS 17 已弃用，仅 warning，建议后续改为 `onChange(of:initial:_:)`。

### #2 智能调整按钮失效 — ✅ PASS（已修编译阻断）
- `TrainingRepository.swift:154-166` 真实 `fetchRecentRecords(limit:)` Core Data 查询实现正确。
- **初版问题**：`PlanViewModel.swift` 末尾 `extension TrainingRepository` 残留存根（重复声明 + 引用不存在的 `trainingRepository` 符号），导致整个 App 编译失败。
- **本轮修复**：已删除该 9 行存根；唯一真实调用点 `PlanViewModel.swift:150` 走 `trainingRepository` 属性 → 真实实现，不受影响。lint 0 错误。

### #3 挤压技术图标空白 — ✅ PASS
- `TrainingModels.swift:18` 已改为 `case .squeeze: return "rectangle.compress.vertical"`（iOS 17 部署目标可用）。

### #4 密码/FaceID 冷启动失效 — ✅ PASS
- 改动① `ControlTrainingApp.swift:53` — `AppState.init()` 末尾 `isLocked = SecurityService.shared.isSecurityEnabled()`（方法定义于 `SecurityService.swift:227`，逻辑正确）。
- 改动② `ContentView.swift:127-130` — `LockScreenView.onAppear` 中 `guard authMode != .none` 自动放行，防止竞态卡死。
- 改动③ `AppDelegate.swift:23-25` — `applicationWillEnterForeground` 移除冗余 `authenticateIfNeeded()` 调用（认证统一由 onAppear 管理）。

> 轻微提示（非阻断）：`configureProtection()` 注释与现实现已不符（仍写"由 applicationWillEnterForeground 处理"），属遗留注释，不影响运行。
