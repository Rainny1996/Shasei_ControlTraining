### 🟠 严重（功能异常或数据风险）

> *(已修复于 v2.0)*

#### BUG-CT-03：`SecurityService` 与 `CryptoService` 使用独立的加密密钥

> *(已修复于 v2.0)*

### 🟡 中等（代码质量问题）

#### BUG-CT-04：`DataController` 文件保护重复设置
#### BUG-CT-05：`TrainingRecord.selfRating` 无边界校验
#### BUG-CT-06：AppDelegate 音频会话配置缺少 `allowBluetooth` 的 `mixWithOthers`

> *(以上均已修复于 v2.0)*

### 🔵 建议（优化项）

| 编号 | 问题 | 文件:行号 | 建议 |
|------|------|----------|------|
| S01 | 全部中文硬编码，无国际化 | 全局 | 使用 `NSLocalizedString` + `Localizable.strings` |
| S02 | `print()` 代替日志框架 | 全局 | 使用 `os_log` 或 `OSLog` |
| S03 | `NotificationService` 时间重复 | NotificationService.swift:114-146 | 提取工厂方法 |
| S04 | `AppState` UserDefaults key 字符串散落 | ControlTrainingApp.swift:72-77 | 提取为枚举常量 `UserDefaultsKey` |
| S05 | `KeychainService` 与 `SecurityService` 各自内嵌 Keychain 操作 | — | 复用 KeychainService（⚠️ v2.0 修复引入回归，v3.0 修复） |
| S06 | `CryptoService.encryptObject/decryptObject` 内部两次 Base64 编码 | — | 优化编码次数 |
| S07 | `ViewModel` 中使用 `DispatchQueue.main.async` | HomeView.swift:172 | 使用 `@MainActor` |
| S08 | `PlanView` 直接访问 `appState` 而非通过 ViewModel | PlanView.swift:39,41 | 统一状态管理 |
