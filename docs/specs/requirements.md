# 需求文档：男性控制训练 App（iOS）

> **文档状态**：v2.2（陪练增强版）
> **最后更新**：2026-07-12
> **适用范围**：iOS 原生 App（SwiftUI），纯本地离线运行
> **说明**：本文档为开发与设计验收的基线。所有验收标准均带唯一编号（AC-x.y），便于代码审查反向追溯需求覆盖率。

---

## 1. 项目概述

**项目名称**：男性控制训练 App（内部代号 `ControlTraining`）
**目标平台**：iOS（iPhone 优先，详见 §4 技术约束）
**目标用户**：中年男性（35–55 岁）
**核心目标**：通过科学、系统的训练方法，帮助用户安全、渐进地提升射精控制能力。

**目标用户特征 → 设计约束**（新增）：
- 该年龄段常见视力变化（老花），界面须默认支持放大字号与高对比度；
- 操作路径须简短（关键操作 ≤ 2 次点击到达）；
- 文案须通俗、避免歧义，医学术语需配简短解释。

## 2. 文档约定

### 2.1 验收标准编号
格式：`AC-<需求号>.<序号>`，例如 `AC-6.1` 表示需求 6 的第 1 条验收标准。

### 2.2 量化约定（消除模糊词）
| 模糊描述（禁用） | 替换为可测量描述 |
|------------------|------------------|
| 流畅的动画 | 动画帧率 ≥ 60fps，无明显掉帧 |
| 快速找到 | 目标入口 ≤ 2 次点击到达 |
| 直观展示 | 指定具体组件（如日历热力图） |
| 及时/较快 | 指定时限（如响应 ≤ 200ms） |

### 2.3 评分量表统一定义
- 自评/难度类量表：**1–5 分**（1=极差/极易，5=极好/极难），超出范围视为非法输入。
- 能力综合评分：**0–100 分**（见需求 6）。

## 3. 术语表（Glossary）

| 术语 | 定义 |
|------|------|
| 射精控制 | 通过训练延长性刺激耐受、自主延迟射精的能力 |
| 凯格尔运动 | 通过收缩盆底肌（PC 肌）增强控制力的训练 |
| 停-动技术（Start-Stop） | 在临近射精临界点时停止刺激、待消退后继续的节律训练 |
| 挤压技术（Squeeze） | 在临界点时轻捏冠状沟处以抑制射精反射的技术 |
| 呼吸训练 | 配合深慢呼吸降低交感神经兴奋、延后射精的训练 |
| 骨盆底肌训练 | 针对盆底肌群的系统性收缩/放松训练 |
| 陪练模式 | App 提供语音引导与计时节奏的实时训练陪伴 |
| 打卡 | 用户标记当日训练任务已完成的行为 |
| 复盘 | 训练后填写简短问卷并记录心得的回顾行为 |

## 8. 需求明细

### 需求 1 - 训练方法介绍

**用户故事**：作为用户，我需要了解各种射精控制训练方法的详细介绍，以便选择并正确执行适合自己的训练。

**功能描述**：提供射精控制领域常见且科学的训练方法介绍，涵盖方法原理、操作步骤、注意事项、禁忌人群与来源标注，帮助用户全面了解并选择。

**验收标准**：
- **AC-1.1** 展示训练方法分类列表，至少包含：凯格尔运动、停-动技术、挤压技术、呼吸训练、骨盆底肌训练。
- **AC-1.2** 每个方法包含：训练原理、操作步骤（图解或动画）、注意事项、**禁忌人群（AC-C.5）**、预期效果、适用人群、来源标注（AC-C.2）。
- **AC-1.3** 训练方法按难度分级：初级 / 中级 / 高级，可在列表筛选。
- **AC-1.4** 支持收藏功能，收藏入口 ≤ 2 次点击到达（AC-NF 量化约定）。
- **AC-1.5** 训练内容基于医学文献，每个方法展示来源/参考文献（AC-C.2）。
- **AC-1.6** 「图解」采用内置矢量动画或静态图资源（资源随包内置，不依赖网络）。

---

### 需求 2 - 实时陪练功能

**用户故事**：作为用户，我需要在训练过程中获得实时语音引导和节奏提示，以便正确执行训练并保持节奏。

**实现决策（已定）**：
- 语音引导采用 **系统 TTS（AVSpeechSynthesizer）** 实时合成中文（zh-CN）提示语，保证文案可配置、零音频素材维护成本；
- 关键节点（准备开始、训练结束）辅以**预录短音效**增强反馈；
- 音频会话统一在 `AudioService` 配置（`.playback` + `.spokenAudio` + `.duckOthers` + `.allowBluetooth` + `.mixWithOthers`），避免多处重复配置（既有 BUG-CT-06）。

**验收标准**：
- **AC-2.1** 进入陪练模式提供倒计时准备阶段（默认 5s，可配置 3–10s）。
- **AC-2.2** 训练中提供 TTS 语音提示：动作开始、节奏调整、休息提醒、训练结束。
- **AC-2.3** 提供可视化计时器，显示当前训练阶段与剩余时间，帧率 ≥ 60fps（AC-NF.2）。
- **AC-2.4** 支持多种节奏模式：基础模式、渐进模式、间歇模式（各模式阶段时长可配置）。
- **AC-2.5** 训练中支持暂停 / 继续，暂停时 TTS 与计时同步挂起。
- **AC-2.6** 训练结束自动记录数据（时长、完成度、训练类型）并写入复盘（需求 5）。
- **AC-2.7** 支持后台音频播放，锁屏后语音引导不中断（后台模式 `audio` 已启用）。
- **AC-2.8** 提供呼吸引导动画，与训练节奏同步；动画帧率 ≥ 60fps。
- **AC-2.9** 音频会话被来电/其他 App 抢占时，训练计时暂停并提示用户手动恢复（明确中断行为）。
- **AC-2.10** 用户在陪练中强制退出 App，须生成**部分训练记录**（标记 `completed=false`），保证复盘数据一致性（关联需求 5）。

### 需求 3 - 个人计划制定

**用户故事**：作为用户，我需要根据自身情况获得个性化训练计划，以便有针对性地提升控制能力。

**实现决策（已定）**："智能生成"为**本地规则引擎**（非 AI/云端），依据评估问卷答案映射计划参数。

**初始评估问卷字段**（AC-3.1）：
年龄区间、当前能力自评（1–5）、训练经验（无/少量/规律）、身体状况（有无相关疾病/手术史）、训练目标（延时/控制力/综合）。

**评估 → 计划映射规则（草案，可校准）**：

| 输入维度 | 取值 | 映射结果 |
|----------|------|----------|
| 能力自评 | 1–2 | 初始频率 3 次/周，单次 10min，难度以初级为主 |
| 能力自评 | 3–4 | 初始频率 4 次/周，单次 15min，初+中级混合 |
| 能力自评 | 5 | 初始频率 5 次/周，单次 20min，中+高级混合 |
| 训练经验 | 无 | 首周仅凯格尔+呼吸（基础适应） |
| 训练经验 | 规律 | 可直接引入停-动/挤压技术 |
| 目标 | 延时 | 停-动/挤压技术权重提高 |
| 目标 | 综合 | 五类方法均衡排布 |

**验收标准**：
- **AC-3.1** 新用户完成初始评估问卷（上述 5 字段），必填项缺失不可进入主页。
- **AC-3.2** 根据评估结果按上述映射规则生成计划：每日训练项目、训练时长、训练频率、阶段目标。
- **AC-3.3** 计划按周期组织：短期（1 周）/ 中期（1 月）/ 长期（3 月），各周期目标可见。
- **AC-3.4** 支持手动调整计划中的训练项目与参数（增删/改时长）。
- **AC-3.5** 计划完成进度可视化（进度条 + 已完成/总任务数）。
- **AC-3.6** 根据训练数据（完成度、自评、打卡率，见需求 5/6）动态调整后续周计划，调整需有记录可查。
- **AC-3.7** 支持设置训练提醒通知（系统本地通知，非推送）。
- **AC-3.8** 提供 ≥ 3 套计划模板供快速选择（如"新手入门""延时专项""综合提升"）。

### 需求 4 - 每日打卡

**用户故事**：作为用户，我需要每日打卡记录训练完成情况，以便养成持续训练的习惯。

**定义约定**（新增）：
- 「有效打卡」= 当日至少完成 1 次训练（陪练完成或手动标记完成），部分记录（AC-2.10）不计入有效打卡。
- 「本月打卡率」= 当月有效打卡天数 / 当月计划训练天数 × 100%。

**验收标准**：
- **AC-4.1** 首页展示当日训练任务与打卡状态。
- **AC-4.2** 完成训练后可打卡，记录打卡时间（本地时区）。
- **AC-4.3** 连续打卡天数统计与展示（断签后归零）。
- **AC-4.4** 打卡日历视图（热力图），连续天数高亮标识。
- **AC-4.5** 打卡成功给予激励反馈（动画 + 鼓励语），动画 ≥ 60fps。
- **AC-4.6** 支持补签功能，默认每月限 3 次（次数可配置），补签记录标记来源。
- **AC-4.7** 打卡统计：总天数、连续天数、本月打卡率（按上述公式）。
- **AC-4.8** 跨日边界：以设备本地时区 00:00 为日期切换点；可配置是否允许"前一天补卡"。

### 需求 5 - 复盘功能

**用户故事**：作为用户，我需要对训练过程进行回顾分析，以便了解进步情况和改进方向。

**验收标准**：
- **AC-5.1** 训练结束后弹出简短复盘问卷：自我感受评分（1–5）、训练难度（1–5）、身体反应（多选标签）。
- **AC-5.2** 历史训练记录列表：日期、类型、时长、完成度、自我评分。
- **AC-5.3** 训练数据趋势图表：能力变化曲线、训练频率统计（本地图表组件）。
- **AC-5.4** 支持周复盘与月复盘报告生成；报告可手动触发，并在每周一/每月 1 日本地自动提示生成。
- **AC-5.5** 复盘报告含：训练总结、进步亮点、待改进项、下期建议（建议可一键加入计划，关联需求 3）。
- **AC-5.6** 支持添加文字备注记录训练心得（纯本地）。
- **AC-5.7** 评分范围强制 1–5，越界视为非法（呼应既有 `selfRating` 边界校验）。

### 需求 6 - 当前状态分析（核心算法草案）

**用户故事**：作为用户，我需要了解自己当前的控制能力水平和改善趋势，以便调整训练策略。

**分析模型（草案，可校准）**：
综合控制能力评分 `S`（0–100）采用**加权多维度模型**：

```
S = Σ(w_i × D_i)，权重 Σw_i = 1
```

| 维度 D_i | 权重 w_i | 数据来源（滚动窗口：最近 30 天） | 计算方式（草案） |
|----------|----------|----------------------------------|------------------|
| 持久力 | 0.25 | 累计训练时长、单次最长时长 | `min(100, 累计时长/目标时长×60 + 单次最长/目标单次×40)` |
| 控制力 | 0.25 | 计划完成率、复盘自评均值 | `完成率×50 + 自评均值(1–5 映射为 0–100)×50` |
| 恢复力 | 0.20 | 连续打卡天数、中断次数 | `min(100, 连续天数/14×100) × (1 - 中断惩罚)` |
| 呼吸配合 | 0.15 | 启用呼吸引导的训练占比 | `呼吸引导训练次数 / 总训练次数 × 100` |
| 肌肉力量 | 0.15 | 凯格尔/骨盆底类训练频率与累计时长 | 同持久力思路，仅统计该类训练 |

> 目标时长/目标单次为可配置常量（默认：周目标 60min、单次目标 20min）。权重与阈值集中配置，便于 A/B 校准。

**等级划分**：
| 分数区间 | 等级 |
|----------|------|
| 0–20 | 入门级 |
| 21–40 | 初级 |
| 41–60 | 中级 |
| 61–80 | 高级 |
| 81–100 | 专家级 |

**验收标准**：
- **AC-6.1** 基于训练数据按上述模型计算综合评分（0–100），模型参数可配置。
- **AC-6.2** 五维能力雷达图（持久力/控制力/恢复力/呼吸配合/肌肉力量），各维 0–100。
- **AC-6.3** 能力等级按上表划分并展示。
- **AC-6.4** 历史评分变化趋势折线图（按周聚合）。
- **AC-6.5** 根据当前状态给出个性化改善建议（文本，由最低维驱动）。
- **AC-6.6** 识别薄弱环节（最低分维度）并推荐对应训练（关联需求 1/3）。
- **AC-6.7** 每周一 00:00 本地时间自动重算；数据不足（< 3 次训练）时显示"暂无足够数据"，不显示虚假评分。
- **AC-6.8** 算法须有单元测试覆盖（给定输入可断言输出），否则无法验收（呼应测试体系）。

### 需求 7 - 隐私保护（技术实现）

**用户故事**：作为用户，我需要确保个人数据和训练记录得到充分保护，以便安心使用。

**验收标准**：
- **AC-7.1** 支持 Face ID / Touch ID 验证解锁。
- **AC-7.2** 本地数据加密存储（AES-256-GCM，统一由 `CryptoService` 单一密钥入口，消除既有双密钥风险 BUG-CT-03）。
- **AC-7.3** 应用切后台时自动模糊界面（scenePhase 变更触发遮罩）。
- **AC-7.4** App 图标与名称不暗示敏感内容（AC-C.4）。
- **AC-7.5** Core Data 数据库排除 iCloud 备份（`addskipBackupAttribute` 或 `cloudKit` 关闭）。
- **AC-7.6** 密钥仅存 Keychain（`kSecAttrAccessibleWhenUnlockedThisDeviceOnly`）。

### 需求 8 - 数据管理（备份 / 导出 / 彻底删除）【新增】

**用户故事**：作为用户，我需要掌控自己的数据归属，以便在换机或退出时安全备份或清除数据。

**验收标准**：

### 需求 10 - 自定义训练计划

**用户故事**：作为用户，我希望不局限于系统生成的计划或固定模板，能够自己挑选训练方法、安排训练日期，或从模板出发再做调整，并把满意的组合保存为「我的模板」以便复用。

**实现决策（已定）**：
- 支持两种组建方式：**① 选模板再改**（预填该模板的方法与排期后调整）；**② 空白自建**（无预填，自由选目标/难度/方法/排期）。两者共用同一编辑器，仅初始数据源不同。

- 「我的模板」归类为本地用户数据，由 Core Data 持久化（与计划数据一致），排除 iCloud 备份（呼应 AC-7.5/AC-NF.7），支持复用与删除。
- 同一训练日可安排多种方法（呼应 Q3，支持「一日多方法」）；自定义阶段以「按日分组」为单位管理各日的方法集合。

**验收标准**：
- **AC-10.1** 提供「自定义计划」入口（计划页菜单或无计划页按钮），≤ 2 次点击到达（呼应 AC-NF 量化约定）。
- **AC-10.2** 支持「选模板再改」：选定预设模板后编辑器预填其方法与排期，可在此基础上增删/替换方法、调整训练天数与具体日期。
- **AC-10.3** 支持「空白自建」：无预填内容，可先选目标/难度，再自由挑选训练方法与排期。
- **AC-10.4** 自定义阶段可调维度限于「方法、每周训练天数、具体日期」三类，不得暴露时长/强度/周期调整入口（避免与需求 11 职责重叠）。模式选择依附于「方法」选择（选定方法后一并选其模式，见需求 13 AC-13.7），不视为新增独立可调维度。
- **AC-10.5** 可将自定义结果保存为「我的模板」（名称由用户自定义），写入 Core Data 持久化；模板库同时展示预设模板与「我的模板」，支持复用与删除。
- **AC-10.6** 生成计划写入当前活跃计划；覆盖已有计划前须二次确认（明确提示"将替换当前计划"）；写入后进度按 `TrainingPlan.updateProgress()` 规则重算并刷新视图。
- **AC-10.7** 自定义界面满足 Dynamic Type（AC-NF.4）、可点击区域 ≥ 44×44 pt（AC-NF.5）、关键元素带 VoiceOver 标签（AC-NF.6）。

### 需求 13 - 方法专属训练模式与逐动作语音指导

**用户故事**：作为用户，我希望陪练的训练模式能针对不同训练方法提供专业、细分的节奏方案（而非全局统一的"基础/渐进/间歇"），并且在训练过程中每个动作都能得到与其内容相符的语音提示，而不是只有笼统的"收缩/放松"。

**背景与动机**：
- 现有 `TrainingMode` 枚举为全局三项（基础/渐进/间歇，见 `TrainingModels.swift`），与具体训练方法的专业节奏脱节。例如凯格尔的"慢速收缩"与"快速收缩"、停-动技术的"多次暂停"、挤压技术的"快速挤压"等，在专业训练中本就是不同模式，但当前无法区分选择。
- 现有语音仅有关键词"收缩/放松/休息"（见需求 2 实现与 `AudioService.announceContract/Relax/Rest`），无法体现每个动作的具体要领（如"收缩骨盆底肌，保持三秒，向上向内提起""暂停，深呼吸，让兴奋度下降"），引导性弱。

**实现决策（草案，待评审）**：
- 新增领域模型 `MethodMode`（方法模式）与 `ModeActionStep`（动作步骤），由 `TrainingMethod.trainingModes` 持有，实现**数据驱动**的方法专属模式目录。
- `CoachViewModel` 的阶段序列直接由所选 `MethodMode.steps` 循环铺满 `defaultDuration` 生成，每步携带独立 `voiceInstruction`/`breathInstruction`，天然实现"逐动作语音"。
- **兼容策略**：保留 `TrainingMode` 枚举作为遗留字段（被大量源码/测试引用），新增 `modeId`/`modeName` 承载方法专属模式；`AnalysisService` 按 `modeName` 聚合（回退 `mode.rawValue`）。

**方法专属模式目录（基于专业训练方法推导，与 `TrainingContentData` 各方法说明一致；语音文案仅作动作引导，不含医疗诊断）**：

#### 凯格尔运动（初级为主）
| 模式 | 难度 | 节奏要点 | 典型动作步骤（显示标签 / 语音文案 / 时长） |
|------|------|----------|--------------------------------------------|
| 慢速收缩 | 初级 | 等长持久力 | 慢缩3s（"收缩骨盆底肌，向上向内提起，保持三秒"）→ 放松3s（"缓慢放松，让肌肉完全松弛"），循环 |
| 快速收缩 | 初级 | 快速反应 | 快缩1s（"快速收缩，立即提起"）→ 放松2s（"放松，准备下一次"），循环 |
| 阶梯式收缩 | 中级 | 30%→60%→100% 阶梯 | 轻收30% 5s → 中收60% 5s → 全收100% 5s → 分阶放松，循环 |

#### 停-动技术（中级为主）
| 模式 | 难度 | 节奏要点 | 典型动作步骤 |
|------|------|----------|--------------|
| 标准停-动 | 中级 | 刺激~2min→暂停30s→降至5级 | 自我刺激（"保持舒适节奏，专注感受兴奋上升"）→ 识别临界（"兴奋度7-8级，准备暂停"）→ 暂停深呼吸30s（"停止刺激，深吸气4秒，屏2秒，呼6秒"）→ 重新刺激 |
| 多次暂停 | 中级 | 刺激1min→暂停20s | 刺激1min → 暂停20s深呼吸 → 循环更多次 |
| 渐进缩短暂停 | 高级 | 逐步缩短暂停/延长刺激 | 刺激逐渐延长、暂停逐渐缩短，挑战阈值 |

#### 挤压技术（中级为主）
| 模式 | 难度 | 节奏要点 | 典型动作步骤 |
|------|------|----------|--------------|
| 标准挤压 | 中级 | 挤压15-20s→释放→恢复 | 刺激→识别临界→挤压15-20s（"拇指与食指对向轻压冠状沟，适度力度，保持"）→释放放松→重新刺激 |
| 快速挤压 | 中级 | 短挤压~10s | 刺激→快速挤压~10s→释放 |
| 轻压维持 | 高级 | 轻压维持抑制 | 临界时轻压维持，降低强度但持续 |

#### 呼吸训练（初级为主）
| 模式 | 难度 | 节奏要点 | 典型动作步骤 |
|------|------|----------|--------------|
| 腹式呼吸 | 初级 | 吸4-呼6 | 吸气4s（"用鼻缓慢吸气，感受腹部隆起"）→ 呼气6s（"缓慢呼气，感受身体放松"） |
| 4-2-6 呼吸法 | 初级 | 吸4-屏2-呼6 | 吸气4s → 屏气2s（"轻轻屏住"）→ 呼气6s |
| 呼吸+收缩结合 | 中级 | 呼气收缩-吸气放松 | 呼气时收缩盆底3s（"呼气同时提起盆底肌"）→ 吸气放松（"吸气，肌肉松开"） |

#### 骨盆底肌综合训练（高级为主）
| 模式 | 难度 | 节奏要点 | 典型动作步骤 |
|------|------|----------|--------------|
| 耐力训练 | 高级 | 10s收缩-10s放松 | 收缩保持10s（"全力收缩，保持稳定，正常呼吸"）→ 放松10s，循环 |
| 阶梯式收缩 | 高级 | 30/60/100% | 轻30% 5s → 中60% 5s → 全100% 5s → 分阶放松 |
| 快速反射 | 高级 | 1s快收-2s放松×20 | 快收1s（"快速全力收缩"）→ 放松2s，连做20次 |
| 功能性模拟 | 高级 | 模拟兴奋上升-全力收缩-放松 | 轻收30%→升60%→模拟临界全收100% 5s→慢放松，循环 |

**跨需求依赖（实现顺序约束）**：本需求依赖 需求 10（自定义计划须承载 per-method 模式——`DayDraft`/`UserPlanTemplateDay` 结构升级见上方实现要点）与 需求 11（逐条编辑须可携 mode 字段）；当前 需求 10 的 `buildCustomPlan` 尚未填写 mode 字段，须由任务 22 补齐后本需求方可端到端生效。

**验收标准（草案 AC，待评审确认）**：
- **AC-13.1** 每种训练方法提供**方法专属**的训练模式集合（不再使用全局统一的"基础/渐进/间歇"），五方法模式目录与上表一致，且每个模式至少包含 2 个动作步骤。
- **AC-13.2** 模式选择入口随所选训练方法动态变化：切换方法时，可选模式列表同步刷新（覆盖陪练准备页、陪练首页、自定义计划编辑器、计划项详情入口）。
- **AC-13.3** 每个模式由若干「动作步骤」组成，每步含：动作类型（收缩/放松/暂停/呼吸/刺激等，用于配色与图标）、显示标签、独立语音文案、时长（秒）、可选呼吸引导。
- **AC-13.4** 陪练过程中按动作步骤**逐条播报语音**：不同动作对应不同语音文案（如"收缩骨盆底肌，向上向内提起，保持三秒"），替代原有通用"收缩/放松/休息"提示。
- **AC-13.5** 语音播报与当前动作阶段、剩余时间、可视化计时器同步显示；呼吸引导动画随步骤的呼吸阶段联动。
- **AC-13.6** 陪练从「今日训练动作」（需求 12）或自定义计划项进入时，默认预选该计划项/模板所选的模式。
- **AC-13.7** 自定义计划（需求 10）的每「方法」可指定一个训练模式（**mode 为 per-method，而非 per-day**），写入 `PlanItem` / 「我的模板」并随计划带入陪练（落库字段 `modeId`/`modeName`）。模式选择依附于「方法选择」——选定某方法后一并选定其模式，视为方法选择的子步骤，不新增独立可调维度（呼应 AC-10.4）。
- **AC-13.8** 模式选择界面与逐动作提示满足 Dynamic Type（AC-NF.4）、可点击区域 ≥ 44×44 pt（AC-NF.5）、VoiceOver 标签（AC-NF.6）。
- **AC-13.9** 训练记录与分析按所选「模式名」聚合（兼容遗留 `TrainingMode` 字段，空则回退其 rawValue），不破坏需求 6 分析。
- **AC-13.10** 模式目录与动作语音文案**基于专业训练方法**，与 `TrainingContentData` 各方法说明一致；文案仅作动作引导，不含医疗诊断（呼应 AC-C.2/AC-C.5）。

**实现要点（供开发参考，详见设计规格与 tasks.md）**：
- 数据：`TrainingModels.swift` 新增 `MethodMode`/`ModeActionStep`；`TrainingMethod` 增 `trainingModes`；`TrainingRecord` 增 `modeId`/`modeName`；`TrainingActionPhase` 扩展分类。
- 陪练：`CoachViewModel` 按所选 `MethodMode.steps` 生成 `phaseSequence`；`CoachView`/`TrainingPreparationView`/`PlanItemDetailView` 模式选择改为方法专属并带入 `initialMode`；`VoiceGuideService` 新增 `announceActionInstruction(_ step)`。
- 计划：`DayDraft`/`UserPlanTemplateDay` 由扁平 `methodIds:[UUID]` **升级为**按方法的结构化数组 `methodSelections:[(methodId:UUID, modeId:UUID?)]`（per-method 模式，保留「一日多方法」Q3，不推翻需求 10 按日分组结构）；`PlanItem` 增 `modeId`/`modeName`；`PlanService.buildCustomPlan`/`draftFromTemplate`/`draftFromUserTemplate` 由 `methodSelections` 生成带 mode 的 `PlanItem`；`PlanBuilderView` 每方法选中后提供方法专属模式选择（默认首个），`PlanViewModel` 透传。
- 分析：`AnalysisService` 按 `modeName` 聚合。
